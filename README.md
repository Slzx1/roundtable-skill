# 圆桌会议系统 (Roundtable)

让多位蒸馏人物围绕一个话题进行多轮讨论交流。任何通过女娲（huashu-nuwa）生成的技能都可以直接参与会议，零配置。

## 使用方式

### 基本用法

```
开个会，让马克思和毛泽东讨论一下「技术进步是否必然导致失业？」，2轮
```

### 参数说明

- **主题**：你想让他们讨论的问题或话题
- **参与者**：用中文名、简称或英文名指代，系统自动匹配到对应技能
- **轮次**：讨论轮数，默认 2 轮。一轮 = 每个参会者各发言一次

### 会议流程

1. **开场发言**：每位参与者独立发表核心观点（不计入轮次）
2. **讨论轮次**：逆序轮流发言，每人每轮回应上一人的观点
3. **结语**：每位参与者总结最终立场（不计入轮次）

用户可以在任意阶段插话、提问，或输入「继续」推进。

### 快捷命令

| 输入 | 效果 |
|------|------|
| `继续` | 推进到下一轮讨论 |
| `进入结语` | 提前结束讨论，进入结语环节 |

## 输出文件

每场会议会在 `transcripts/` 下生成三份文件：

| 文件 | 内容 |
|------|------|
| `transcript.md` | 完整原始对话记录，保留每个人的原文 |
| `summary.md` | 结构化摘要：各方观点、共识与分歧、开放问题 |
| `outline.md` | 讨论大纲：立场脉络、议题结构、会议统计 |

## 如何让新人物参与

只要通过女娲（huashu-nuwa）蒸馏了新人物技能，并安装到以下任一位置即可：

- 项目级：`.claude/skills/<name>-perspective/`
- 用户级：`~/.claude/skills/<name>-perspective/`

下次开会时直接使用该人物的名字即可，无需任何额外配置。

## 会议助理原则

会议助理（本技能）严格遵守以下原则：
- 只管理流程，不参与讨论内容
- 不评价任何人的观点
- 不引导讨论方向
- 传话时传递原文，不总结不转述
- 不报时——只报阶段和轮次号
